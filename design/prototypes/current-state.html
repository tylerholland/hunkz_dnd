Issue 1: No field is visually distinguished as required. "Character Name" is the only field that blocks saving (the handleCreate guard checks !charData.name) but it looks identical to optional fields like Alias, Alignment, or Origin. Users discover the requirement only when they click "Save Character" and hit an alert dialog — a disruptive, after-the-fact error pattern.
Combat & Spells
4
Issue 4: The label says "comma-separated" but doesn't explain that each item becomes a separate tagged chip on the character sheet. Users familiar with comma use in prose (e.g., spell descriptions) may not realize commas are the functional separator. Spell names could also hypothetically contain commas. The input gives no preview of how the chips will render, so the format and outcome are both opaque.
