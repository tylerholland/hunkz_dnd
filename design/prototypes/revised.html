Portrait
💬

This caption is only visible when a portrait is uploaded. If you add one now, it will appear once you upload an image.

3 Fix applied: Field renamed from "Portrait Tagline" to "Portrait Caption" to match how users think of image captions. A persistent inline note makes the visibility constraint explicit — no hidden state. The user now knows upfront that caption + portrait go together. (Alternate approach would be to always show the tagline in view mode even without a portrait — chosen fix here prioritizes the more common case where portraits and captions are paired, and avoids orphaned italic text at the top of a sheet with no image context.)
Identity
Fields marked * are required to save the character. All others are optional and can be filled in later.
1 Fix applied: A single amber asterisk (*) marks Character Name as required. A one-line legend above the form explains the convention and reassures the user that everything else can be filled in later. No other field is marked — keeping required indicators rare preserves their signal strength. The legend text avoids words like "mandatory" or "must" in favor of a lighter tone ("can be filled in later") that matches the app's voice.
Combat & Spells
Separate multiple spells with a semicolon — e.g. Hunter's Mark; Misty Step; Pass Without Trace
Preview — how this will appear on your sheet
4 Fix applied: Three changes together solve the confusion. (1) Separator changed from comma to semicolon — unambiguous, not used in D&D spell names. (2) Placeholder and helper text both show the separator in context and mention "or just one spell" so the field doesn't feel like it demands a list. (3) A live preview panel appears as the user types, showing the chips exactly as they will look on the sheet — eliminating the save-and-check loop. The preview is labeled "how this will appear on your sheet" so its purpose is immediately clear.

2 Fix applied: A persistent helper line sets the only expectation that matters: any password is valid, and it's needed to edit later. No minimum length is enforced (matching the actual backend behavior), so no false requirements are implied. If the user types 1–2 characters, a gentle amber advisory appears (not an error, not a blocker) suggesting a longer password would be easier to remember. This advisory disappears once they type more characters. The overall tone is informative rather than gatekeeping.

Swift as shadow, silent as snow

3 Fix applied (view mode): The caption is now rendered unconditionally at the top of the sheet header area, in a centered italic block, regardless of whether a portrait is present. When a portrait is present it remains beneath the image as before; when no portrait exists it appears as a standalone atmospheric line — still readable and meaningful. This ensures the user's written content is never silently lost. (The alternative — keeping it portrait-gated but warning explicitly in the edit field — was rejected because hiding user-authored content is worse than showing it in a slightly different position.)
